Types d'Ordres

Les différents types d'ordres.

10 min Débutant

🎯 Introduction: L\

Vous avez la bonne analyse, le bon timing, le bon niveau - mais si vous ne savez pas COMMENT entrer et sortir correctement, vous laissez de l'argent sur la table.

Les ordres sont vos OUTILS d'exécution. Chaque type a son utilité. Utiliser le mauvais ordre au mauvais moment peut transformer un trade gagnant en perdant.

Sur les Futures, avec leur haute volatilité et liquidité, le choix de l'ordre est particulièrement crucial.

📊 SCHÉMA: Les 3 ordres fondamentaux

┌─────────────────────────────────────────────────────────────────────────┐
│              LES 3 TYPES D'ORDRES FONDAMENTAUX                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  1. MARKET ORDER (Ordre au marché)                                       │
│  ════════════════════════════════                                        │
│  • Exécution: IMMÉDIATE au meilleur prix disponible                     │
│  • Garantie: Exécution garantie, prix NON garanti                       │
│  • Slippage: Possible, surtout en haute volatilité                      │
│  • Usage: Quand l'exécution est PLUS importante que le prix            │
│                                                                          │
│       Prix                                                               │
│        5002 ─── Ask (vendeurs)                                          │
│        5001 ─── Spread                                                  │
│        5000 ─── Bid (acheteurs)                                         │
│                                                                          │
│  Market BUY → Vous achetez à 5002 (le Ask)                              │
│  Market SELL → Vous vendez à 5000 (le Bid)                              │
│                                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  2. LIMIT ORDER (Ordre limite)                                           │
│  ════════════════════════════════                                        │
│  • Exécution: UNIQUEMENT au prix spécifié ou meilleur                   │
│  • Garantie: Prix garanti, exécution NON garantie                       │
│  • Slippage: Aucun (vous avez le prix demandé ou rien)                  │
│  • Usage: Entrées patientes, niveaux précis                             │
│                                                                          │
│  Limit BUY à 4995 → Vous n'achetez QUE si le prix descend à 4995       │
│  Limit SELL à 5010 → Vous ne vendez QUE si le prix monte à 5010        │
│                                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  3. STOP ORDER (Ordre stop)                                              │
│  ════════════════════════════════                                        │
│  • Exécution: Devient MARKET quand le prix touche le niveau stop        │
│  • Usage principal: Stop-loss pour limiter les pertes                   │
│  • Slippage: Possible car devient market                                │
│                                                                          │
│  Stop SELL à 4990 → Si le prix touche 4990, vente market immédiate     │
│  Stop BUY à 5010 → Si le prix touche 5010, achat market immédiat       │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

🔧 Ordres avancés pour les Futures

┌─────────────────────────────────────────────────────────────────────────┐
│              ORDRES AVANCÉS                                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  4. STOP-LIMIT ORDER                                                     │
│  ════════════════════════════════                                        │
│  • Combinaison: Stop + Limit                                            │
│  • Devient LIMIT (pas market) quand le stop est touché                  │
│  • Avantage: Évite le slippage du stop market                           │
│  • Risque: Peut ne pas s'exécuter si le prix "gap" au-delà             │
│                                                                          │
│  Ex: Stop-Limit SELL, Stop=4990, Limit=4988                             │
│  → Si prix touche 4990, ordre limit à 4988 est placé                    │
│  → Si le marché gap à 4980, l'ordre n'est PAS exécuté!                 │
│                                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  5. OCO (One Cancels Other)                                              │
│  ════════════════════════════════                                        │
│  • 2 ordres liés: si l'un s'exécute, l'autre est annulé               │
│  • Usage: Stop-loss + Take-profit en même temps                         │
│                                                                          │
│  Ex: Position LONG à 5000                                               │
│  ├─ Stop SELL à 4990 (protection)                                       │
│  └─ Limit SELL à 5020 (take profit)                                     │
│  → Si TP touché, stop annulé. Si stop touché, TP annulé.               │
│                                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  6. BRACKET ORDER                                                        │
│  ════════════════════════════════                                        │
│  • Entrée + Stop + Target en un seul ordre                              │
│  • Automatise la gestion complète du trade                              │
│  • Recommandé pour tous les traders                                     │
│                                                                          │
│  Ex: Bracket LONG                                                        │
│  ├─ Entrée: BUY LIMIT 5000                                              │
│  ├─ Stop: SELL STOP 4990                                                │
│  └─ Target: SELL LIMIT 5020                                             │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

📋 GUIDE: Quand utiliser quel ordre?

┌─────────────────────────────────────────────────────────────────────────┐
│              QUAND UTILISER QUEL ORDRE?                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌─────────────────────┬───────────────────────────────────────────┐    │
│  │     SITUATION       │           ORDRE RECOMMANDÉ               │    │
│  ├─────────────────────┼───────────────────────────────────────────┤    │
│  │ Entrée sur niveau   │ LIMIT - Prix précis, pas de slippage     │    │
│  │ Sortie urgente      │ MARKET - Exécution immédiate             │    │
│  │ Stop-loss           │ STOP ou STOP-LIMIT                       │    │
│  │ Take-profit         │ LIMIT - Prix garanti                     │    │
│  │ Breakout trading    │ STOP BUY/SELL - Entrée sur cassure       │    │
│  │ Trade complet       │ BRACKET - Tout automatisé               │    │
│  │ Scalping rapide     │ MARKET - Vitesse critique                │    │
│  │ Swing patient       │ LIMIT - Patience pour meilleur prix      │    │
│  └─────────────────────┴───────────────────────────────────────────┘    │
│                                                                          │
│  ⚠️ CAS PARTICULIERS:                                                   │
│                                                                          │
│  • Pendant les NEWS (FOMC, NFP): ÉVITEZ les market orders               │
│    → Slippage de 5-20 ticks possible!                                   │
│                                                                          │
│  • Marchés ILLIQUIDES (overnight, pre-market):                          │
│    → Utilisez des limit orders ou attendez plus de liquidité            │
│                                                                          │
│  • Haute VOLATILITÉ:                                                     │
│    → Stop-limit plutôt que stop market pour éviter le slippage         │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

⚡ Slippage: Comprendre et minimiser

┌─────────────────────────────────────────────────────────────────────────┐
│              LE SLIPPAGE EXPLIQUÉ                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  DÉFINITION: Différence entre le prix attendu et le prix d'exécution   │
│                                                                          │
│  EXEMPLE:                                                                │
│  • Vous placez un MARKET BUY quand le prix affiché est 5000             │
│  • Votre ordre est exécuté à 5002                                       │
│  • Slippage = 2 points = $100 sur ES                                    │
│                                                                          │
│  CAUSES DU SLIPPAGE:                                                     │
│  ├─ Haute volatilité (news, open, close)                                │
│  ├─ Faible liquidité (overnight, lunch)                                 │
│  ├─ Taille de position trop grande                                      │
│  └─ Ordres market en conditions extrêmes                                │
│                                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  COMMENT MINIMISER LE SLIPPAGE:                                          │
│                                                                          │
│  ✅ Tradez pendant les heures de haute liquidité (US session)           │
│  ✅ Utilisez des limit orders quand possible                            │
│  ✅ Évitez les market orders pendant les news                           │
│  ✅ Gardez des positions proportionnelles à la liquidité                │
│  ✅ Sur ES: max ~10 contrats par ordre pour éviter l'impact             │
│  ✅ Utilisez stop-limit au lieu de stop market si volatilité élevée    │
│                                                                          │
│  💡 SLIPPAGE ACCEPTABLE SUR ES:                                          │
│  • Normal: 0-1 tick ($0-12.50)                                          │
│  • News: 2-5 ticks ($25-62.50)                                          │
│  • Crash/Spike: 10+ ticks ($125+) - ÉVITEZ!                             │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

📝 CAS PRATIQUE: Exécution complète d\

┌─────────────────────────────────────────────────────────────────────────┐
│              EXÉCUTION D'UN TRADE - STEP BY STEP                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  CONTEXTE:                                                               │
│  • Signal MIA: LONG ES sur niveau GEX à 5000                            │
│  • Stop recommandé: 4992 (-8 points)                                    │
│  • Target recommandé: 5020 (+20 points)                                 │
│  • Prix actuel: 5003                                                     │
│                                                                          │
│  ÉTAPE 1: PRÉPARER LE BRACKET ORDER                                      │
│  ├─ Entrée: BUY LIMIT à 5000 (au niveau GEX)                            │
│  ├─ Stop: SELL STOP à 4992                                              │
│  └─ Target: SELL LIMIT à 5020                                           │
│                                                                          │
│  ÉTAPE 2: PLACER L'ORDRE                                                │
│  • L'ordre est en attente, le prix est à 5003                           │
│  • Patience: on attend que le prix descende à 5000                      │
│                                                                          │
│  ÉTAPE 3: EXÉCUTION                                                      │
│  • 16h15: Le prix touche 5000                                           │
│  • BUY LIMIT exécuté à 5000 ✅                                           │
│  • Stop et Target sont maintenant ACTIFS automatiquement               │
│                                                                          │
│  ÉTAPE 4: GESTION                                                        │
│  • Prix monte à 5010: On ne touche à RIEN                               │
│  • Prix atteint 5020: SELL LIMIT exécuté automatiquement               │
│  • Stop à 4992 est ANNULÉ automatiquement (OCO)                         │
│                                                                          │
│  RÉSULTAT:                                                               │
│  • Profit: +20 points × $50 = +$1,000                                   │
│  • Slippage: ZÉRO (limit orders)                                        │
│  • Intervention manuelle: AUCUNE après le setup initial                 │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘

❌ ERREURS COMMUNES avec les ordres

  • •❌ ERREUR #1: Market orders pendant les news → Slippage de 10+ ticks possible
  • •❌ ERREUR #2: Oublier de placer le stop → Un "je le ferai après" = position sans protection
  • •❌ ERREUR #3: Limit order trop ambitieux → "Je veux 5000 pile" quand le low est 5002
  • •❌ ERREUR #4: Stop trop serré → 2 ticks de stop = touché par le bruit normal du marché
  • •❌ ERREUR #5: Ne pas utiliser de bracket orders → Gérer manuellement = erreurs garanties
  • •❌ ERREUR #6: Taille d'ordre trop grande → Impact sur le prix et slippage accru
  • •❌ ERREUR #7: Modifier les ordres en panique → Le plan est fait AVANT, pas pendant

✅ POINTS CLÉS à retenir

  • •✅ MARKET = Exécution garantie, prix non garanti
  • •✅ LIMIT = Prix garanti, exécution non garantie
  • •✅ STOP = Protection, devient market quand touché
  • •✅ STOP-LIMIT = Stop sans slippage, mais peut ne pas s'exécuter
  • •✅ BRACKET = Entrée + Stop + Target en un ordre - RECOMMANDÉ
  • •✅ Le slippage est réel - utilisez des limit orders quand possible
  • •✅ Pendant les news: ÉVITEZ les market orders
  • •✅ Automatisez votre exécution pour éliminer les erreurs humaines
